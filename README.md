# E-Commerce-Sales-Customer-Behavior-Analysis

## Project Overview

This project focuses on analyzing ecommerce sales and customer behavior data using SQL Server and Power BI. The project covers data cleaning, data modeling, SQL analysis, KPI generation, and interactive dashboard development to generate meaningful business insights.
---
## Problem Statement
Ecommerce businesses generate large amounts of sales and customer data, making it challenging to identify meaningful trends and business insights.

This project aims to analyze ecommerce transaction data using SQL Server and Power BI to track sales performance, customer behavior, product trends, and payment patterns through an interactive analytics dashboard.

## Dataset Information

### Dataset Details
- Dataset Name: E-Commerce Customer Behavior and Sales Dataset
- Source: Kaggle
- Description: Turkish online retail platform transactional data (2023–2024) containing customer behavior, sales, and engagement information.
- Total Rows: 17,050
- Total Columns: 18

The dataset includes customer demographics, purchasing behavior, product categories, payment methods, device usage, delivery details, and sales-related metrics used for business analysis and dashboard reporting.

## Data Modeling
A Star Schema model was created using:
- Fact Table: fact_orders
- Dimension Tables:
  - dim_customers
  - dim_products
  - dim_date
  - dim_payment
  - dim_device

## Key Business Insights
- Electronics category generated the highest revenue.
- Returning customers contributed significantly to total sales.
- Credit Card was the most frequently used payment method.
- Mobile devices had the highest order volume.
- Peak sales activity occurred on specific days of the month.
- Major cities generated the highest sales contribution.
